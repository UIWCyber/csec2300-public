# Handout H01: CIA Triage Cards

**CSEC 2300 Foundations of Cyber Security | Dr. Gonzalo D Parra**
**Used with Lecture 1: Course Launch | In-Class Activity: CIA Triage (think, pair, share)**

## Instructions

1. **Solo (2 minutes).** Read the six incident headlines below. Tag each one with the CIA property or properties it violates: **C** (Confidentiality), **I** (Integrity), and/or **A** (Availability).
2. **Pairs (3 minutes).** Compare answers with a partner. Most incidents hit at least two properties. If you tagged only one, find the second one.
3. **Debrief (5 minutes).** Be ready to defend your tags when cold-called. The point is the triad as an analysis lens, not trivia.

## The Six Headlines

| # | Headline | C | I | A |
|---|----------|---|---|---|
| 1 | Attackers steal a retailer's customer database containing 40 million payment records |   |   |   |
| 2 | A botnet floods a regional bank's online portal, knocking it offline during business hours |   |   |   |
| 3 | Hacktivists replace a city government's homepage with a political manifesto |   |   |   |
| 4 | A departing employee emails the company's confidential product roadmap to a competitor |   |   |   |
| 5 | Ransomware encrypts a hospital's patient records; staff cannot open a single file |   |   |   |
| 6 | An intern force-pushes over the main branch and three weeks of commit history vanish |   |   |   |

**Tip:** Ask three questions of every incident. Did someone see data they should not see (C)? Did data change or disappear in a way that breaks trust (I)? Did a system or its data become unusable for legitimate users (A)?
