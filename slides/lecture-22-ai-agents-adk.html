Chatbot vs Agent

  • Chatbot: text in, text out. It can describe how to reset a password but cannot do it.
  • Agent: perceives a goal, DECIDES on a step, ACTS through a tool, reads the result, and loops until done.
  • An agent has four parts: a model (the brain), tools (the hands), memory (the notes), and a loop (the orchestration).
DefinitionAI agent = a system that uses a language model to choose and take actions toward a goal, not just to produce text.
AnalogyA chatbot is a librarian who tells you where a book is. An agent is an assistant who walks over, pulls the book, and puts it on your desk.

The Agent Loop, Step by Step

  • The core pattern is called ReAct: the model Reasons, then Acts, then Observes, then loops.
  1. 1Reason: the model thinks about the goal and decides the next step.
  2. 2Act: the model asks to run a tool, with specific arguments.
  3. 3Observe: your code runs the tool and hands the result back to the model.
  4. 4Repeat: the model reads the result and either takes another step or gives the final answer.
DefinitionReAct = a loop of Reasoning and Acting where the model interleaves thinking with tool calls until it is done.

Tool / Function Calling - How Agents Touch the World

  • You describe each tool to the model: its name, what it does, and its parameters (a schema).
  • The model emits a structured call, for example read_note(name="todo"), as data, not as running code.
  • Your code executes that call and returns the result; the model reads it and continues.
  • Crucial and scary: the MODEL decides which tool and what arguments, using text it was handed, which may be attacker-controlled (recall indirect injection, L21).
DefinitionTool calling (function calling) = giving the model a menu of functions it can request, so its output can trigger real actions in your code.

Multi-Step Tool Use

  • Real goals chain many calls. "Book me a flight" becomes search, compare, select, pay, confirm.
  • Each step feeds the next; the model plans across the whole chain.
  • More steps plus more autonomy equals more capability AND more places to go wrong.

What ADK Gives You

  • ADK (Agent Development Kit) is a framework to define an agent, register tools, and run the loop with logging and tracing built in.

A Tool Is Just a Python Function

  • A tool is an ordinary function with a clear name, typed arguments, and a docstring the model reads.
A minimal, safe tool
$ cat tools.py
def read_note(name): # tool the model may request
path = SAFE_DIR / name # scoped to one sandboxed folder
return path.read_text() # returns text back to the model
  1. 1def read_note(name) names the tool and its one argument. The model sees this signature.
  2. 2SAFE_DIR / name scopes every read to one folder. The tool cannot wander the whole disk.
  3. 3return path.read_text() hands the file contents back as the observation for the next loop step.
TipThe docstring you write becomes the tool's description to the model. Vague docstrings cause wrong tool calls.

Registering and Running the Agent

  • You hand ADK the tools and a plain-English instruction, then run it against a goal.
Wire up a two-tool agent
$ python agent.py "What time is it and what is in my todo note?"
[trace] reason: I need the time, then the note
[trace] act: get_time() -> 14:05
[trace] act: read_note(name="todo") -> "email the TA"
[trace] answer: It is 14:05 and your todo note says: email the TA
  1. 1The instruction you set was: "Use tools to answer; never invent data." That single line steers behavior.
  2. 2Each [trace] line is one loop step: reason, act, act, then answer.
  3. 3Notice the model called two tools in the right order with no code from you telling it to.

Tracing = Accountability for Agents

  • Every tool call is logged: which tool, what arguments, what result. That is the agent's audit trail.
  • This is AAA accounting (L1, L6) applied to AI: without traces you cannot debug, attribute, or investigate an incident.
  • Capstone requirement: your agent's actions are logged and reviewable in the wiki.
Why it mattersWhen an agent does something wrong, "the AI did it" is not an answer. The trace is how you find out what actually happened.

Every Tool Is a Granted Privilege

  • Giving an agent a tool = giving it that capability under model control. A file-read tool can read ANY file it can reach.
  • Least privilege for agents: the narrowest tools, scoped paths, read-only where possible, and no run_shell unless truly required.
  • Sandbox the tools: your M4 containers are the natural jail. Agent tools run in a locked-down container, not on your host.
Watch outAdding a run_shell(cmd) tool hands the model your entire operating system. One injected instruction can then run anything. Do not add it lightly.

Key Takeaways

  • Agent = model + tools + memory + a reason/act/observe loop. It DOES things, not just says them.
  • Tool calling lets model output trigger real actions, and the model chooses from untrusted text.
  • Trace everything; treat every tool as a privilege granted under least-privilege discipline.
  • Capability and risk rise together. Next session is about keeping the human in charge.
