What Cryptography Buys You

  • Crypto is not one tool. It delivers three of the security properties you already know.
  • Confidentiality through encryption, integrity through hashes, authentication and non-repudiation through signatures.
  • What it does NOT buy: availability, bug-free code, or good key management. Those are still on you.
DefinitionCryptography = using math to protect information so that only intended parties can read or trust it.
DefinitionPlaintext = the readable message. Ciphertext = the scrambled version an attacker sees. Encryption turns one into the other.

One Rule Before Anything Else

  • The security must live in the secret key, never in keeping the algorithm secret.
  • Real algorithms (AES, RSA) are public and studied by thousands of experts. That scrutiny is why we trust them.
DefinitionKerckhoffs's principle = a system stays secure even if everything about it except the key is public knowledge.
Watch outNever roll your own crypto. Home-made ciphers fail in ways you will not see until an attacker does. Use vetted libraries.

How Symmetric Works

  • One key both encrypts and decrypts. It is fast and handles large amounts of data easily.

The Catch - Sharing the Key

  • Symmetric is great once both sides have the key. Getting the key to the other side is the hard part.
  • AES (block cipher) is today's standard; ChaCha20 (stream cipher) is common on phones.
Watch outYou cannot send the key over the same hostile network you are trying to protect. If an attacker is listening, they get the key too.

The Key-Distribution Problem

  • With many people talking privately in pairs, the number of shared keys explodes.
  • For n people, you need n(n-1)/2 separate keys. Ten people already need forty-five keys. It does not scale.

  • The breakthrough: two mathematically linked keys, one you publish and one you hide.

Public Key and Private Key

  • Each person has a public key (share freely) and a private key (guard with your life). They are linked but hard to reverse.
  • For confidentiality: encrypt with the recipient's PUBLIC key; only their PRIVATE key can decrypt.
  • For authenticity: sign with your PRIVATE key; anyone verifies with your PUBLIC key. That gives non-repudiation.

RSA and ECC - You Already Use These

  • RSA uses large keys; ECC (elliptic curve) gives the same strength with much smaller keys.
  • You have been using asymmetric crypto all semester: your SSH keys and your signed Git commits.

The Best of Both

  • Asymmetric is roughly a thousand times slower than symmetric: too slow for bulk data.
  • So use asymmetric just once, to agree on a random session key, then encrypt all the real traffic with fast symmetric AES.

What a Hash Does

  • A hash function takes any input and returns a fixed-size digest. It is deterministic, fast, and one-way.
  • The avalanche effect: flip a single bit of input and about half the output bits change.

Compute a Hash in the Terminal

Hash a string and watch the avalanche
$ echo "hello" | sha256sum
5891b5b522d5df086d0ff0b110fbd9d2... -
$ echo "Hello" | sha256sum
66a045b452102c59d840ec097d59d9467... -
  1. 1echo "hello" prints the text; the | pipe feeds it into the next program.
  2. 2sha256sum is the program: compute the SHA-256 digest of its input.
  3. 3The - at the end just means "the input came from the pipe, not a file."
  4. 4Two nearly identical inputs, two completely different digests. Integrity checks rely on exactly this.

Hashing Is Not Encryption

  • Encryption is reversible if you hold the key. Hashing has no key and no way back to the original.
  • Uses: verifying downloads, deduplication, Git commit IDs, and password storage.
DefinitionHMAC = a hash combined with a shared secret key. It proves both that a message is unchanged (integrity) and who sent it (authenticity).

You Already Trust Hashing - Git

  • Every Git object ID is a hash of its contents. Change one byte of history and every downstream ID changes.
Git's integrity is hashing
$ git cat-file -p HEAD | head -2
tree 4b825dc642cb6eb9a060e54bf8d69288fbee4904
parent 9fceb02...
  1. 1git cat-file -p HEAD prints the latest commit object.
  2. 2The tree and parent lines are themselves hashes pointing at other objects.
  3. 3This chain of hashes is why Git history is tamper-evident. You have been using crypto since week three.

Broken Hashes - Exam Bait

  • Not every hash is safe. MD5 and SHA-1 are broken for security: attackers can craft collisions.
  • Use SHA-256 (or the SHA-2/SHA-3 family) for anything that matters.
Watch outIf a tool or exam question offers MD5 or SHA-1 for security, that is the wrong answer. Collisions against both are demonstrated and practical.

Passwords - A Special Case of Hashing

  • Never store passwords as plaintext. Never store them as a plain fast hash either.
  • GPUs try billions of SHA-256 guesses per second, so a fast hash of a leaked password falls quickly.
DefinitionSalt = a random per-user value added before hashing, so identical passwords get different digests and precomputed tables fail.
DefinitionSlow (memory-hard) hash = a function like bcrypt, scrypt, or argon2, deliberately expensive to compute, making mass guessing impractical.
CSEC 2300 Foundations of Cyber Security1.0 General Security Concepts

The Breach Math

  • Plaintext storage: a breach is instant total loss.
  • Fast unsalted hash: rainbow tables crack common passwords in seconds.
  • Salted argon2: an attacker needs years per strong password. That gap is your whole defense.

Key Takeaways

  • Symmetric is fast but has the key-sharing problem; asymmetric solves distribution but is slow; hybrid uses both and is everywhere.
  • Hashing is one-way integrity; passwords additionally need salt plus a slow hash.
  • You already operate crypto daily through SSH, Git, and HTTPS. Now you can name the parts.
