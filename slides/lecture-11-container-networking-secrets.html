Recap - Image, Container, Isolation

  • You met containers only one session ago, so let us re-anchor the core words before we network them.
  • An image is a frozen blueprint of an app and its files. A container is one running copy of that image.
  • Each container is isolated: it has its own filesystem and its own view of the network, kept apart from the host and from other containers by the Linux kernel.
DefinitionContainer = a running, isolated instance of an image. It behaves like a lightweight private computer, but shares the host's kernel instead of running a full separate operating system.
AnalogyThink of each container as a sealed shipping container on one cargo ship (the host). Today's lecture is about which containers get a hallway to each other, and which stay sealed.

First, the Words We Need

  • We use three networking words today. Let us define them before anything else, because you have barely seen networks yet.
DefinitionPort = a numbered door on a computer, like 80 or 8000. One program listens behind each door for incoming connections.
DefinitionNetwork (here) = a private virtual switch Docker creates so containers plugged into it can talk to each other.
DefinitionPublishing a port = poking a hole from the host machine to a container, so the outside world can reach that door.
AnalogyA container is an apartment. A network is the hallway apartments share. Publishing a port is cutting a door from the hallway to the street.

The Default Bridge - One Big Hallway

  • When you start containers without saying otherwise, Docker plugs them all into one default network called the bridge.
  • Every container on that bridge can reach every other one. That is convenient and also a security problem.

Publishing a Port - Facing the World

  • Inside a network, containers reach each other. The outside world cannot, until you publish a port.
Publish a container port to the host
$ docker run -p 8080:80 nginx
  1. 1-p is the publish option.
  2. 28080 (left) is the port on your host machine, the street-facing door.
  3. 380 (right) is the port inside the container, where nginx listens.
  4. 4A browser visiting host port 8080 is forwarded to container port 80.
Watch outThe moment you publish a port, that container faces the world. Publish only what truly needs outside access.

"Can Reach" Should Equal "Needs to Reach"

  • Flat networks are exactly how intrusions spread: break one container, then hop to every neighbor.
  • The security goal is simple to say: a container should be able to reach only what it actually needs.
DefinitionLateral movement = an attacker, after breaking into one machine, moving sideways to others it can reach. Flat networks make this trivial.

User-Defined Networks

  • You can create your own named networks and attach each service only to the ones it needs.
Create two isolated networks
$ docker network create frontend
$ docker network create backend
  1. 1docker network create makes a new private virtual switch.
  2. 2frontend and backend are just names you choose.
  3. 3A container attached only to backend cannot be reached from frontend. That gap is your security boundary.
DefinitionSegmentation = dividing a system into zones so a breach in one zone does not automatically reach the others. Also called microsegmentation between services.

Why This Stops an Attacker

  • Open WebUI needs two things: browser access (publish port 3000) and the Ollama API (backend network).
  • Ollama needs nothing but calls from WebUI: backend only, never published to the host.
  • Attack analysis: if WebUI is compromised, the attacker still cannot reach the Docker host network or expose the model API to the internet.

One File Declares the Whole Stack

  • A compose file is one YAML document listing every service, network, volume, and published port.
Build the whole world from one file
$ docker compose up -d
[+] Running 3/3 webui Started ollama Started db Started
  1. 1docker compose up reads docker-compose.yml and starts everything in it.
  2. 2-d means detached: run in the background and give your terminal back.
  3. 3One command, the entire multi-container stack, wired exactly as written.
DefinitionYAML = a plain-text format for structured settings, using indentation instead of braces. Compose files are written in it.

Why Compose Is a Security Win

  • The compose file is reviewable, diffable, and version-controlled: your network architecture goes through pull-request review like any code.
  • It cannot silently drift from reality the way a whiteboard diagram does; the file is what runs.
TipIf you cannot point to the line in the compose file that publishes a port, that port should not be published.

Volumes - What Persists, What Leaks

  • Containers are disposable; their internal files vanish on removal. Volumes keep data that must survive.
  • A named volume is Docker-managed storage (chat history, model files). A bind mount maps a host folder straight into the container.
Watch outBind mounts expose host paths to the container. Mounting a sensitive host directory is a classic misconfiguration. Mount the minimum, and use read-only (:ro) wherever you can.

Why Env Vars Leak

  • Environment variables feel easy, so they are sadly common. They are also the weakest choice.
  • They show up in docker inspect, in the /proc filesystem, in application logs, and in crash dumps.
Watch outA secret in an env var is one docker inspect away from exposure. Prefer a mounted secret file with tight permissions, or an external manager (Vault, cloud KMS) for rotation and audit.

Demo - Build the Segmented Stack

  • Instructor runs docker compose up -d for a three-service stack (webui, api, db) with frontend and backend networks from one file.
See who is on which network
$ docker network inspect backend | grep Name
"Name": "api"
"Name": "db"
  • From the WebUI container, wget -qO- api:8000 works: they share a network, and Docker resolves the name api.
  • From a frontend-only container, reaching db fails with a timeout. That failure is the proof the wall holds.
  • Finally, a compose secrets: block mounts a db password file; docker inspect shows no secret in the environment.

Security Angle - Lateral Movement, Contained

  • An attacker's playbook after a first foothold: map the network, then pivot to soft internal targets.
  • Segmentation turns one breach into one breached segment, and buys your detection tools time to work.
  • The same pattern scales everywhere: VLANs in enterprises, VPCs in the cloud, networks in compose.
