What Is the Shell?

  • The shell is the program that reads what you type and asks the operating system to run it.
  • On Linux and Mac the common shell is called bash or zsh. On the class servers it is bash.
DefinitionShell = the program that turns your typed commands into actions the operating system performs.
AnalogyThe shell is a waiter. You (the customer) give an order in words. The waiter takes it to the kitchen (the operating system) and brings back the result.

Anatomy of the Prompt

  • Before you type anything, the shell shows a prompt. It is telling you useful facts.
student@server:~/labs$
  1. 1student is your username: who you are logged in as.
  2. 2server is the machine name: which computer you are on. This matters when you connect to remote servers.
  3. 3~/labs is where you are right now (your current folder). ~ means your home folder.
  4. 4$ means "the shell is ready for your command." A # here would mean you are the all-powerful root user.

Anatomy of a Command

  • Every command has the same three-part shape. Learn to see the parts and no command looks scary.
One command, three parts
$ ls -l /home/student
  1. 1ls is the program: the thing you want to run (here, "list files").
  2. 2-l is an option (also called a flag): it changes how the program behaves (here, "long detailed format").
  3. 3/home/student is the argument: the thing the program acts on (here, which folder to list).
TipOptions usually start with a dash. One dash for short options (-l), two dashes for long ones (--all).

Reading a Command Out Loud

  • Practice turning symbols into plain English. This is the single most useful skill today.
Say what each command does
$ cd /var/log# change directory into the system log folder
$ ls -a# list all files, including hidden ones
$ cp notes.txt backup.txt# copy notes.txt to a new file named backup.txt
$ rm old.txt# remove (delete) the file old.txt
Watch outrm deletes immediately. There is no recycle bin in the terminal. Read twice, run once.

Absolute vs Relative Paths

  • A path is an address for a file or folder. There are two kinds.
  • Absolute paths start at the root / and give the full address. They work from anywhere.
  • Relative paths start from where you are standing right now.
Both commands open the same folder, from home
$ cd /home/student/labs# absolute: the complete address
$ cd labs# relative: labs inside where I already am
Note. means "here", .. means "the folder above me". cd .. moves up one level.

Reading Output and Errors Without Panic

  • The terminal talks back. Two kinds of replies: normal output, and errors.
  • Errors are not scary. They are the computer telling you exactly what went wrong.
An error is a clue, not a failure
$ cd projekts
bash: cd: projekts: No such file or directory
  1. 1Read it right to left: "No such file or directory" is the problem.
  2. 2projekts is the thing it could not find. Look again: it is a typo for projects.
  3. 3The fix is almost always in the message. Slow down and read it.

Copy-Paste Hygiene

  • You will copy commands from lab instructions. Do it safely.
  • Never paste a command you do not understand, especially anything with sudo or rm.
  • Watch for a leading $ in instructions: that is the prompt symbol, not part of the command. Do not copy it.
Watch outPasting commands from random websites is a real attack path. In this course, only paste from the lab instructions and the course hub.

Key Takeaways

  • The shell runs your typed commands. The prompt tells you who and where you are.
  • Every command is program + options + arguments. Read it in those three parts.
  • Paths are addresses: absolute starts at /, relative starts from here.
  • Errors are clues. Read the message before you retry.
