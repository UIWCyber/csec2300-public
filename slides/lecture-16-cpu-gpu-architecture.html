What the CPU Is Built For

  • Design goal: latency, which means finishing one chain of instructions as fast as humanly possible.
  • It has a handful of large, complex cores (8 to 24 is typical) loaded with clever tricks.
  • Great at branching logic and operating-system work; poor at doing ten thousand identical things at once.
DefinitionLatency = how long one single task takes from start to finish. A low-latency chip answers one question very quickly.
DefinitionCore = one independent worker inside a processor that can execute an instruction stream on its own.

The Tricks Inside a CPU Core

  • Pipelining: start the next instruction before the last one fully finishes, like an assembly line.
  • Branch prediction and speculative execution: guess which way an if will go and start early.
  • Big caches: keep recently used data physically close so the core does not wait on slow memory.
AnalogyA CPU core is a master chef. Only a few of them in the kitchen, but each one is fast, flexible, and handles a complicated order end to end.
NoteRemember speculative execution. Next session it comes back as the root of the Spectre and Meltdown side-channel flaws.

What the GPU Is Built For

  • Design goal: throughput, which means pushing the most total work through per second.
  • Thousands of small, simple cores apply the same operation to a huge pile of data at once.
  • It began life shading millions of pixels per frame; the exact same math runs neural networks.
DefinitionThroughput = total amount of work finished per unit time, even if any single task is not especially fast.
DefinitionGPU (Graphics Processing Unit) = a processor with thousands of simple cores built to do the same operation on massive data in parallel.
AnalogyIf the CPU is a few master chefs, the GPU is a stadium full of line cooks who all crack one egg at the same instant. Terrible for a complex order, unbeatable for ten thousand identical eggs.

Why Neural Networks Love GPUs

  • Running an LLM is mostly enormous matrix multiplications: rows and columns of numbers multiplied and added.
  • Every one of those multiply-add operations is independent, so all the cores can run at once.
  • A CPU can run a small model, just slowly; a GPU makes a local model fast enough to feel interactive.
DefinitionMatrix multiplication = a grid of numbers combined with another grid by many small multiply-and-add steps. It is the core operation inside a neural network.
DefinitionInference = using an already-trained model to produce an answer. This is what happens every time you send a prompt.

The Speed vs. Size Tradeoff

  • Fast memory is tiny and expensive; big memory is cheap and slow. You cannot have all three.
  • So computers stack layers: a little bit of very fast memory, backed by a lot of slow memory.
  • Each step down is roughly ten times slower but a thousand times bigger.
DefinitionCache = a small pool of very fast memory that holds recently or frequently used data so the processor does not wait on slower memory.
DefinitionLocality principle = programs tend to reuse the same data and nearby data, so keeping "hot" data in fast memory pays off.

Why VRAM Is the Line in the Sand

  • A GPU keeps model weights in its own fast memory called VRAM, separate from system RAM.
  • If the weights do not fit in VRAM, performance falls off a cliff or the model refuses to load.
  • Rough rule: a 7-8 billion parameter model at 4-bit quantization needs about 4-5 GB; a 70B model needs 40+ GB.
DefinitionVRAM (Video RAM) = the GPU's own dedicated memory. Model weights must fit here to run fast.
Watch outSystem RAM and VRAM are different pools. "I have 32 GB of RAM" does not mean your 8 GB graphics card can load a big model.

Quantization - Squeezing Models to Fit

  • Quantization stores each weight with fewer bits: 16-bit down to 4-bit, sometimes lower.
  • Fewer bits means a smaller, faster model with only a modest quality loss.
  • This is the single reason a consumer graphics card can run a real large language model at all.
DefinitionQuantization = reducing the number of bits used to store each model weight, trading a little accuracy for much smaller memory use.
AnalogyQuantization is like saving a photo as a smaller JPEG. A little detail is lost, but it fits on the device and looks almost the same.

Reading Your GPU's Memory

  • Before you pick a model, you need to know how much VRAM you actually have. One command tells you.
Check GPU memory and load
$ nvidia-smi
| GPU Name | Memory-Usage |
| 0 RTX 4070 | 1536MiB / 12288MiB |
  1. 1nvidia-smi is the program: the NVIDIA System Management Interface, a status tool for the GPU.
  2. 2It takes no options here, so it prints a full status table for every GPU it finds.
  3. 3In the output, Memory-Usage reads used / total. Here 1536 MiB are in use out of 12288 MiB (12 GB) total.
  4. 4That total number is your budget. It decides the largest model you can load: this card fits a quantized 7-8B model comfortably.
TipNo NVIDIA card? On a Mac, open Activity Monitor and use the GPU pane. On Apple Silicon the GPU shares system RAM, which changes the math (see the trends slide).

GPUs Crack Passwords Too

  • Password cracking is the same parallel math: try billions of guesses, hash each one, compare.
  • A single modern GPU running hashcat guesses on the order of billions of MD5 or SHA-256 hashes per second.
  • This is exactly why argon2 and bcrypt (Lecture 12) are memory-hard: to sabotage GPU parallelism on purpose.
Definitionhashcat = a popular password-cracking tool that uses the GPU to test enormous numbers of guesses in parallel.
Watch outFast hashes like plain SHA-256 are a bad choice for storing passwords precisely because a GPU chews through them. Slow, memory-hard hashes are the defense.

Hardware Trends You Should Recognize

  • Unified memory (Apple Silicon): the CPU and GPU share one pool of RAM, which is why some laptops run surprisingly large models.
  • NPUs and TPUs: chips built only for the matrix math, more efficient than a general GPU.
  • Supply chain: GPU scarcity and export controls make hardware itself a geopolitical security topic.
DefinitionNPU (Neural Processing Unit) = a chip designed specifically for neural-network math, now common in phones and laptops.
NoteOn a unified-memory Mac, "VRAM" and system RAM are the same pool, so a 16 GB Mac can devote more to a model than a 16 GB PC with a small graphics card.

Key Takeaways

  • CPU means latency and a few smart cores; GPU means throughput and thousands of simple cores.
  • The memory pyramid trades speed against size; VRAM is the budget line for local AI.
  • Quantization squeezes real models onto consumer hardware.
  • Every accelerator is dual-use: the defender's AI engine is the attacker's cracking rig.
