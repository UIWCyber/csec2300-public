The God-Token Anti-Pattern

  • An agent needs credentials to act, but which credentials decides the blast radius.
  • Anti-pattern: the agent holds a god-token (admin over everything). One injection equals total compromise.
  • Right pattern: the agent gets its OWN scoped identity, least privilege, per-tool credentials, and short-lived tokens.
DefinitionScoped credential = a permission narrowed to exactly the resources and actions a task needs, and nothing more.
AnalogyYou do not give a house-sitter the keys to the bank, the car, and the safe. You give them one key to one door for one weekend.

The Confused Deputy Problem

  • A privileged agent is tricked by an unprivileged input into misusing its own authority. It is a classic security bug, now AI-shaped.
  • Example: a user (or an injected document) asks the agent to "summarize file X." X is /etc/shadow, which the agent CAN read but the user may NOT.
  • Fix: the agent's authority must be bounded by the REQUESTER's authority, not just its own. Add authorization checks on tool calls.
DefinitionConfused deputy = a program with high privilege that is fooled by a low-privilege actor into using that privilege on the actor's behalf.

Four Ways Autonomy Fails

  • Cascading errors: step 3 builds on a wrong step 2, and no human notices until step 9.
  • Goal misalignment (specification gaming): it optimizes the literal instruction, not your intent ("maximize replies" becomes spam).
  • Runaway loops (LLM10): infinite tool-calling burns tokens and dollars. Set hard limits.
  • Multi-agent amplification: agents prompting agents spread a single injection across the whole system.
Watch outA runaway agent can spend real money or hammer a real API in seconds. Hard caps and step limits are not optional.

Human-in-the-Loop vs Human-on-the-Loop

  • In-the-loop: a human APPROVES before a high-impact action executes (spend money, send external email, delete).
  • On-the-loop: a human MONITORS and can intervene or abort, but routine actions proceed on their own.
  • Out-of-the-loop: full autonomy, acceptable only for low-impact, reversible actions.
DefinitionHuman-in-the-loop (HITL) = a required human approval step inserted before an agent action takes effect.

Designing Approval Gates

  • Classify each tool: reversible and low-impact goes to auto; irreversible or high-impact requires confirmation.
  • Dry-run / preview: the agent proposes the plan; the human approves before execution. This is your git pull-request pattern, applied to actions.
  • Hard backstops independent of the model: rate limits, spending caps, and allow-listed recipients or domains.
TipThe backstops must live OUTSIDE the model. A cap the model can talk itself past is not a control.

Governance & Monitoring for Agents

  • Logging and tracing (L22) feed monitoring: alert on anomalous tool use, volume spikes, and denied-authorization attempts.
  • Kill switch: a way to halt the agent instantly. An OFF button is itself an availability control.
  • Accountability: who owns this agent, and who answers when it errs? "The AI did it" is not an answer.
  • Emerging frameworks to name-drop for the policy paper: NIST AI RMF, EU AI Act risk tiers, and MITRE ATLAS for threats.

Live Demo - Add a Human Approval Gate

  • Take last session's ADK agent and wrap a sensitive tool (write_file) so it PAUSES, prints the proposed action, and requires a typed y/N before executing.
The gate surfaces the action
$ python agent.py# a poisoned note says: delete all files
[trace] act: write_file(path="notes/todo", data="") BLOCKED - awaiting approval
Approve write to notes/todo? [y/N] n
[trace] denied by human; action not executed
  1. 1The injection reached the model, and the model requested the destructive call.
  2. 2The gate turned that request into a human question instead of an action.
  3. 3The human declined, and the action was blocked and logged.
Why it mattersThe model was compromised, and the ARCHITECTURE held. That is the entire point of defense in depth for agents.

Key Takeaways

  • Give agents their own scoped identity, bounded by the requester's authority. This defeats the confused deputy.
  • Autonomy fails via cascades, misalignment, runaway loops, and multi-agent amplification. Bound all four.
  • Match oversight (in/on/out of loop) to impact times reversibility. Gate the dangerous, automate the trivial.
  • When the model is fooled, architecture and human gates are what save you, and someone still OWNS the agent.
