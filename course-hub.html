Hands-On

The 12 Labs

Lab 0 plus 11 graded labs - roughly 60% of your class time is hands-on, and each lab layers a new piece onto your semester-long secure local-AI stack.

Lab 0Environment & identity setup

Configure Git, SSH keys, and your GitHub account, and smoke-test Docker + GPU so nothing blocks you later.

Lab 1Framework mapping

Map a real security scenario onto NIST CSF and CIS Controls to learn how professionals organize defenses.

Lab 2Git & GitHub workflow

Branch, open pull requests, sign commits, and keep secrets out of the repo - collaboration with an audit trail.

Lab 3CI/DevSecOps pipeline

Build a GitHub Actions pipeline that runs SAST, secret, and dependency scanning on every push.

Lab 4Docker hardening

Ship a non-root, minimal-base container image that passes a vulnerability scan with capabilities dropped.

Lab 5Crypto & TLS

Hash, generate key pairs, and TLS-terminate a containerized service with a working certificate.

Lab 6GPU & architecture

Inspect your GPU, run an accelerated workload, and report on TPM / secure boot - hardware as the root of trust.

Lab 7Local LLM stack

Deploy Ollama + Open WebUI in Docker on the GPU and write up the privacy case for local inference.

Lab 8LLM attack & defense

Execute a prompt injection against your own stack, add a guardrail, and prove the mitigation works.

Lab 9Build an ADK agent

Create a tool-using AI agent on a local model - with authorization limits that keep it on a leash.

Lab 10Policy analysis

Analyze your stack against GDPR / HIPAA / FERPA - the written policy paper that assesses privacy directly.

Lab 11Capstone integration

Wire everything into one hardened, CI-gated repository - the final assembly of your semester's work.

Stuck? Use the hint ladder

Every lab ships with three tiers of hints in its README. Climb only as far as you need - hints never change the grader; you always earn what your submission proves.

Tier 1 · Nudge
Points you to the concept and the doc to read. No commands. Little or no point cost.
Tier 2 · Guided
Names the exact tool or flag and the shape of the solution. Modest point cost.
Tier 3 · Near-solution
Shows the command pattern with a blank to fill. Larger point cost - unblocks you without giving the grade away.

How autograding works

  1. Accept the lab through your GitHub assignment repository - you get your own repo from the template.
  2. Work locally, commit, and push. Every push triggers the automated grader (a GitHub Actions workflow).
  3. Read your per-criterion score and feedback in the Actions run - file checks, container checks, TLS handshakes, injection tests.
  4. Revise and push again as many times as you want before the deadline. Your last push counts.

Setup Addendum

Set Up Your Machine

Everything the labs need on your own computer. Full step-by-step instructions live in the Lab 0 assignment on Canvas; this is the quick reference and download list.

Install these (in order)

  1. Git - git-scm.com/downloads (Windows: accept the default options)
  2. Docker Desktop - docker.com/products/docker-desktop (Windows: it will enable WSL 2 for you; say yes and reboot when asked)
  3. Ollama - ollama.com/download (needed from the Local AI module onward)
  4. VS Code (recommended editor) - code.visualstudio.com
  5. GitHub account - github.com/signup (use a professional username; you will show this repo to employers)

Check yourself

Open a terminal (Windows: PowerShell) and run each line. Every one should print a version, not an error:

git --version
docker --version
docker run hello-world
ollama --version

If docker run hello-world fails, Docker Desktop is probably not running - start it and retry. Anything else: bring the exact error message to office hours or post it on the Canvas discussion board. The classroom GPU workstations have all of this pre-installed, so a personal machine is convenient but never required.

The Fine Print, Digested

Policies You'll Actually Ask About

The full versions live in the course outline and syllabus on Canvas - this is the quick reference.

Late work

  • No late work is accepted. Every deliverable - labs, quizzes, papers, and all capstone components - is due at the posted deadline; missing it means a zero for that item.
  • Documented emergency? Contact Dr. Parra before the deadline - accommodations are case by case at the instructor's discretion.
  • Pushing early and often is free insurance: the autograder shows your score before the deadline.

Attendance

  • Up to 3 absences without penalty, excused or not. The 4th absence forfeits the entire 5% attendance grade.
  • Arriving 10+ minutes late counts as tardy; 3 tardies = 1 unexcused absence.
  • Planned university/religious/military absences: written notice at least two weeks ahead. Four or more absences without officially dropping results in a failing grade per university policy.

Academic integrity

Highest standards of academic honesty apply. Cheating and plagiarism are handled per the UIW Student Handbook (Section 5, Part D). In a course about security, integrity is the discipline - signed commits leave an audit trail, and so do you.

This outline is a guideline

The schedule and procedures may change under extenuating circumstances or when it benefits the class. Prominent changes are announced through Canvas Announcements - check it and your university email regularly; that's the official channel.
